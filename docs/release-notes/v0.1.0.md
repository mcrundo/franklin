# Franklin v0.1.0 — Turn technical books into Claude Code plugins

First public release. Franklin reads an EPUB or PDF and produces a full Claude Code plugin — SKILL, reference tree, slash commands, subagents, and plugin packaging — by extracting the book's concepts and workflows into structured data, then generating one artifact at a time.

## Install

```bash
uv tool install franklin-book
# or
pipx install franklin-book
```

Both drop a `franklin` command onto your PATH. The distribution is `franklin-book` on PyPI (since `franklin` was taken); the CLI you type is still `franklin`.

## Quick start

```bash
franklin doctor                             # preflight check: API key, network, disk, etc.
franklin pick                               # interactive .epub/.pdf picker with run-state overlay
franklin run path/to/book.epub              # full pipeline end-to-end
franklin run path/to/book.epub --estimate   # predict cost before spending a cent
franklin run path/to/book.epub --review     # pause between plan and reduce to prune artifacts
```

## Highlights

### Advisor strategy built in (Opus advises, Sonnet executes)

Franklin uses the [advisor pattern](https://algoinsights.medium.com/the-advisor-strategy-how-to-get-claude-opus-intelligence-without-opus-prices-bfd17bbed96b) natively: the `plan` stage uses Claude Opus **once** to design the plugin architecture, and the `reduce` stage uses Claude Sonnet **N times** to execute that plan per artifact. For a typical 28-artifact run that's `1× Opus + 28× Sonnet` instead of `29× Opus` — the same output quality for roughly **20% of the cost**.

`franklin review <run-dir>` is the human-in-the-loop half: pause between plan and reduce, prune artifacts you don't want to pay for, then continue.

### Five-stage pipeline, every stage resumable

```
ingest → map → plan → reduce → assemble
```

Every stage is a standalone CLI command that reads and writes to a run directory, so crashes resume where they stopped and you can replay a single stage after editing a prompt. `franklin run` chains them end-to-end with resume-on-disk semantics.

- **ingest** — EPUB/PDF → normalized chapters. Deterministic, no LLM calls. PDFs get an optional Tier 4 LLM cleanup pass (`--clean`) for mechanical artifact fixes.
- **map** — per-chapter structured extraction (concepts, principles, rules, anti-patterns, workflows, code examples) via Sonnet.
- **plan** — architectural Opus call that designs the plugin layout.
- **reduce** — Sonnet generates each artifact file from the plan.
- **assemble** — `plugin.json`, link/frontmatter/template validators, grade card, `metrics.json`.

### Cost and performance

- **Async Tier 4 cleanup** drops a 29-chapter PDF from ~50 minutes sequential to ~6 minutes at the default `--clean-concurrency=8`.
- **`franklin run --estimate`** predicts per-stage token counts and dollar cost from the parsed `BookManifest` before any paid call. Heuristics lean pessimistic.
- **Live Rich progress bars** on map, reduce, and cleanup show spinner, count, elapsed, ETA, and current item. Final summaries include actual USD spent.

### UX that feels like Claude Code

- **`franklin doctor`** — preflight: Python version, `uv`, Anthropic API key, license state, `claude` CLI, network reachability, disk space. `--skip-network` and `--json` supported.
- **`franklin pick`** — interactive book picker that scans a directory for `.epub`/`.pdf` files and cross-references existing run state (`new` / `partial (stage)` / `assembled (grade)`).
- **`franklin runs list`** — table of every run directory with slug, title, date, last completed stage, and grade.
- **`franklin grade <run-dir>`** — detailed per-artifact rubric breakdown with lowest-grade artifacts and suggested regeneration commands. `--json` for tooling.
- **`franklin review <run-dir>`** — interactive plan pruning. Omit artifacts by index or range (`1,3-5`) before paying for reduce.
- **Resume detection** — `franklin run` on an existing run directory shows per-stage progress and prompts to continue from the first incomplete stage. `--yes` auto-confirms, `--force` restarts.
- **Book metadata confirmation** — detected title and authors are shown after ingest so you can fix wrong EPUB metadata before it turns into plugin identifiers.
- **Friendly error surface** — Anthropic SDK errors (rate limit, auth, 529, connection, timeout) and Franklin errors (missing API key, unsupported format) get classified into actionable blocks with suggested next steps instead of raw tracebacks.
- **"Next steps" block** after every `franklin run` / `franklin assemble` guides you to local install, publish, or iteration commands — no more "what do I do now?"

### Publishing your plugin

```bash
# Try it locally before publishing
franklin install <run-dir> --scope local

# Or persist it to your user or project scope
franklin install <run-dir> --scope user
franklin install <run-dir> --scope project

# When ready to share, publish to a GitHub repository
franklin push <run-dir> --repo owner/name

# Other users install from GitHub
claude plugin install owner/name
```

### License state

v0.1 ships **fully free**. The license module (`franklin license {login, logout, whoami, status}`) is wired in for a future paid tier but the gate is disabled (`_LICENSE_GATE_ENABLED = False`). Every command works without a license.

## Run directory layout

```
runs/<slug>/
  book.json              # BookManifest (evolves across stages)
  raw/chNN.json          # NormalizedChapter — from ingest
  chapters/chNN.json     # ChapterSidecar — from map
  plan.json              # PlanManifest — from plan
  output/<plugin>/       # Generated plugin tree — from reduce/assemble
    .claude-plugin/plugin.json
    skills/<name>/SKILL.md
    references/.../*.md
    commands/*.md
    agents/*.md
  metrics.json           # Grade card output
```

## Requirements

- Python 3.12+
- An Anthropic API key (`ANTHROPIC_API_KEY` env var or OS keychain via `keyring set franklin ANTHROPIC_API_KEY`)
- `claude` CLI on PATH for `franklin install`

## Known limitations

- **PDF support is experimental.** Layout-aware heuristics via pdfplumber catch most cases; tricky layouts (multi-column, scanned pages, unusual fonts) may need manual cleanup. Prefer the EPUB edition when available.
- **Advisor strategy is macro-only right now.** `plan` → `reduce` is the built-in version. Per-artifact Opus briefs before each Sonnet call is a future enhancement gated on grade-card data.
